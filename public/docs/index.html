workspace

A new IDE concept based on web technologies with a focus on extensibility and user experience.

Requirements and Installation

System Requiremets

  • A Modern Browser (Chrome, Firefox, IE Edge, etc)
  • node >5.0 and npm

Installation

To install from NPM (stable) 


npm install -g @cxl/workspace

To install from source (github) 


git clone https://github.com/coaxialsoftware/workspace.git
cd workspace
npm link

Basic Concepts

Workspace

A Workspace is a collection of projects under the same folder or directory. Workspace settings are stored in the optionalworkspace.json file. Multiple instances of workspace can be started for different workspaces.

Project

A project is a directory inside a workspace that contains your source code.

The command :project can be used to open existing projects.

There are many ways to create a project.

Creating a Project from existing sources

Creating a Project from a remote repository

Creating a Project from scratch

Project Files

Project settings are stored in the optional project.json file. This file will override any default settings from workspace.json. You can also use project.local.json if you need to override settings from project.json for your local copy. This is helpful when you dont want to check in those settings in your repository.

Settings

For a list of available settings and descriptions use the :settings command.

Commands

Commands

Use :commands to see a list of commands.

KeyMap

Keymap states are used to associate keyboard events to commands. You can specify the default keymap using the keymap setting.

The ide.keymap.registerState(state, map) function can be used to register a keymap state.

Plugins

Plugins are key components of the IDE.

To see the list of available plugins, use the :plugins command.

Themes

Themes are CSS files. They can be specified by using the theme setting. Default templates are in the public/themes folder.
