Getting Started

System Requiremets

  • A Modern Browser (Chrome, Firefox, IE Edge, etc)
  • node >5.0 and npm

Installation

To install from NPM (stable)

$ npm install -g @cxl/workspace

Starting the Application

$ cd /your/projects/folder $ workspace

Basic Concepts

Workspace

A Workspace is a collection of projects under the same folder or directory. Workspace settings are stored in the optional workspace.json file. Multiple instances of workspace can be started for different workspaces.

Use the workspace.settings command to edit the workspace settings.

Project

A project is a directory inside a workspace that contains your source code.

The command project [path or name] can be used to open existing projects.

Once you open a project, you can use the project.settings command to edit open the project settings file.

There are many ways to create a project.

Creating a Project from existing sources

To create a project from an existing source, just move the project folder to your current workspace.

Creating a Project from a remote repository

Creating a Project from scratch

Using the project command, you will be prompted to create a new project if it currently does not exist.

: project newPath

Project Files

Project settings are stored in the optional project.json file. This file will override any default settings from workspace.json. You can also use project.local.json if you need to override settings from project.json for your local copy. This is helpful when you dont want to check in those settings in your repository.

Commands

Every action in the application has an associated command. Use the commands command to see a list of available commands for your current editor.

KeyMap

Keymap states are used to associate keyboard events to commands. You can specify the default keymap using the keymap setting.

To see a list of keyboard shortcuts for the current editor use the keymap command.

Themes

Themes are CSS files. They can be specified by using the theme setting. Default templates are in the public/themes folder.

Editors

Extensions

To see the list of available extensions, use the plugins command.

Creating Extensions

Extensions are npm packages containing an index.js file and an optional client javascript file.

Installing Extensions

The plugins.install @cxl/workspace.pluginName can be used to install plugins. Use the plugins.uninstall @cxl/workspace.pluginName to remove it.

To install your own custom or unofficial extensions, you can add an entry to the workspace.json file.

{ "plugins": [ "path/to/your/extension/package" ] }

Creating Themes

Themes are CSS Files. You can see a list of available themes in the Plugins list. To use your own custom CSS, specify the path in your project.json or workspace.json theme property.
