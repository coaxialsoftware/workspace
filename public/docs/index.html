workspace

A new IDE concept based on web technologies with a focus on extensibility and user experience.

Overview

  • Easy to install and configure. Runs locally or remotely.
  • Client/Server based.
  • A powerful plugin API and plugins for popular tools.
  • Intelligent Text Editor and Code Assistance.
  • Minimalistic UI. Focus in your code.
  • VIM support.

Requirements and Installation

System Requiremets

  • A Modern Browser (Chrome, Firefox, IE11, etc)
  • node >5.0 and npm

Installation

To install from NPM (stable) 


	npm install -g @cxl/workspace

To install from source (github) 


	git clone https://github.com/coaxialsoftware/workspace.git
	cd workspace
	npm link

Basic Concepts

Workspace

A Workspace is a collection of projects under the same folder or directory. Workspace settings are stored in the optionalworkspace.json file. Multiple instances of workspace can be started for different workspaces.

Project

A project is a folder in a workspace.

Project Files

Workspace looks for a project.json file when initializing a project. This file will override any default settings from workspace.json. You can also use project.local.json if you need to override settings from project.json. This is helpful when you dont want those settings to apply to other users.
